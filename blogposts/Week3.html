Readings:

UI & UX


In the reading, Mailloux explores the relationship between language conventions and the context in the interpretation of language. Alongside with the lecture it highlights how to decide which UI and UX to use, we must first gather how successful it is then it can become a convention.

Mailloux defines convention as suitable practices commonly used in a context. (Mailloux 1983: 399). Indicating how conventions in UI and UX must be agreed upon and used as a base for it to be a convention. When understanding UI and UX conventions as a creator we must look at how the user will react to it and how they feel about it. Thus, the interaction between the user and UI/UX is what is important as that feedback and response is the convention. As if the user can interact with it successfully then it has become the convention but if not, I must go back to the wireframe and find a new UI/UX that can successfully for full it to become the convention. An example is the previous and next buttons, these buttons are at the end of the page, allowing the user to go straight to the next one. It would be ineffective to put them at the top as then the user has to scroll up again to go to the next page.

It shows how UI and UX conventions have become limiting to new ideas. This is because the user becomes used to UI and UX conventions as it may not allow them to experience a new way of how it can be done. Since it is a convention the user will always want something similar to that and may leave your website if it does not meet that convention.
